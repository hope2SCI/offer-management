# 面试复盘 AI 解答功能设计

## 背景

当前面试复盘已经支持记录面试时间、轮次、评价、难度、自我感受、问题记录和备注总结。

下一步目标是让用户基于已记录的面试问题，一键生成参考答案。用户仍然只负责记录面试中遇到的问题，不需要重复写自己的回答。

## 范围

包含：

- 在面试复盘卡片上提供 AI 解答入口。
- 针对单条面试复盘的问题记录生成参考答案。
- 保存、查看、编辑该条复盘的 AI 参考答案。
- 支持重新生成并覆盖当前答案。
- 在头像菜单中增加设置入口，用于配置 Deepseek API Key。
- 支持快速解答和深度思考两种生成模式。

不包含：

- 全站高频问题统计。
- 跨多次面试的 AI 总结。
- 多模型提供商配置。
- AI 答案历史版本。
- 逐题结构化问题卡片。
- 流式输出。

## 用户流程

用户在 `/interviews` 或岗位详情里看到面试复盘卡片。

每张卡片右上角展示 AI 解答按钮：

- 没有答案时显示 `AI 解答`。
- 已有答案时显示 `查看 AI 解答`，并在卡片上展示 `已生成 AI 解答` 状态。

点击按钮后打开 AI 解答弹窗。弹窗展示：

- 原始面试问题记录，只读展示。
- 模式选择：快速解答 / 深度思考。
- 生成按钮或重新生成按钮。
- AI 参考答案编辑区。
- 保存按钮。

如果当前答案已被手动编辑，点击重新生成前需要提示会覆盖现有答案。

## 数据设计

AI 答案直接属于 `InterviewReview`。

新增字段：

- `aiAnswer`：AI 生成或用户编辑后的参考答案。
- `aiAnswerMode`：生成答案时使用的模式，值为 `FAST` 或 `DEEP`。
- `aiAnswerUpdatedAt`：答案最后保存时间。

只保存最新版，不保存历史版本。

## 答案格式

AI 生成结果使用固定文本格式，便于阅读和手动编辑：

```text
问题：
参考答案：
追问/补充点：
```

用户输入的问题记录仍然是自由文本。AI 需要尽量从自由文本中拆分问题；拆分不准确时，用户可以在答案编辑区手动修改。

## AI 设置

头像菜单增加 `设置`。

设置页第一版只包含 Deepseek 配置：

- API Key 输入框。
- 保存按钮。
- 已配置状态。

API Key 属于当前登录用户。前端不回显完整 Key，只展示是否已配置。

第一版提供商固定为 Deepseek，不做提供商切换。Deepseek 的具体模型名、接口地址和参数映射等拿到 API 文档后再落到实现计划中。

## 模式

用户只看到两个模式：

- 快速解答：适合普通技术题和概念题。
- 深度思考：适合系统设计、算法、项目难点和开放题。

模式影响后端调用 Deepseek 时的模型或参数，但不把模型细节暴露给用户。

## 错误处理

- 未配置 API Key 时，AI 解答弹窗提示用户先去设置。
- 面试问题记录为空时，不允许生成答案，提示先填写问题。
- AI 调用失败时，保留现有答案，不覆盖。
- 保存 API Key 失败或保存答案失败时，展示错误信息。

## 验收标准

- 用户可以从面试复盘卡片打开 AI 解答弹窗。
- 用户可以选择快速解答或深度思考。
- 用户可以基于该条复盘的问题生成参考答案。
- 用户可以编辑并保存 AI 参考答案。
- 用户可以重新生成并覆盖当前答案。
- 卡片可以显示是否已生成 AI 解答。
- 用户可以从头像菜单进入设置并保存 Deepseek API Key。
- 未配置 API Key 时不会发起 AI 调用。
